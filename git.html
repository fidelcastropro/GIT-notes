GIT and GITHUB

GITHUB

  • GITHUB is mainly used in versioning.
  • It means we can retrieve all the edited versions from it is created.
  • For Eg:We are creating a webpage and it is running perfectly,Then you are updating the webpage with some updates.After the Updation the webpage not functions as expected.At this situation you can use the previous version which was performed well by using The git and github.

GIT

  • GIT is used to connect the github with the local directory in our system
  • git config --global user.name "fidelcastropro"
  • git connfig --global user.email "fidelcastropro6@gmail.com"
  • git clone repository link - This command will be ran in vscode terminal,The clone of the repository will be made in the folder(location) in which the terminal is executed
  • To add the files created by the user in the editor to the github clone in our PC Follow the given command git add filename
  • The command given in the previous line only adds the file to the Repository clone in our PC but not saving it.To save the file in the Repository Clone git commit -m "message"
  • Now the newly created file will only saved in the repository clone inour PC.
  • To update it in the GitHub Server we have to give the command git push origin main Here the main is the directory name in the github Server.
  • git add filename - The purpose of the command is not adding the file to the clone repository but adding the each and every updates on the file to the repository clone.
  • So git add filename is used to delete,modify the file in the clone repository.
  • In the previous cases we have seen how to create a file in our PC text editor and update it in the repository clone and pushing it to the GitHub server.This case we are going to create the file in the GitHub server but we have to update the changes made in the server to the clone repository in our PC.
  • To do the process mentioned above git pull command is used.
  • In the above given steps we have added files to the Git Repository by creating a Clone in our PC.Now we are going to learn how to add files to the repository without creating a clone in our PC
    • From the previous repository we have to navigate to the new location where we are going to create the new files which are to be saved in the github repository cd .. coming one folder back from the previous location.
    • cd newFolderName to open that folder
    • In the previous cases we have created the clone so the keyword origin points out the repository url. But in this case we have not created the clone so we have to specify the link of the repository to link that new folder with the Repository git remote add origin repositoryLink - repositoryLink is the Repository link of the new repository created.
    • git init - this command initializes that folder to be connected with that repository.
    • git add filename
    • git commit -m "commit message"
    • in the second repository the directory is named as master,so
    • git push origin master
    • git branch -M "desiredFilePathName" - Then the folder is created in the Repository with the name we specified
    • git push origin nameOfTheFolderCreatedinRepository - This command is used to store the files in diffrent folders of the GitHub Repository
    • If more than one folders exist in the repository then the command given in the previous line will be useful to store the files in different folders.
  • git add . - This command will be used to add all the files in that location to the repository
